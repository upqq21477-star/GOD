# 像素拳擊遊戲：完整企劃案

> **彙整範圍：** 渲染架構、假3D人物系統、動作系統、套路神經網路、動態權重系統、低幀率動態捕捉體素運鏡系統（Voxel Cine-System）、即時動態選單戰鬥系統（Instant-Tactics Combat System）

---

## 專案簡介

一款運行於瀏覽器的像素風格拳擊遊戲，核心美學為刻意的低幀率卡頓感，致敬早期街機格鬥遊戲。技術上採用「假3D體素投影」架構，在純2D Canvas環境中製造立體人物的視覺錯覺，以動作捕捉數據驅動人物動作，配合套路神經網路系統管理戰鬥決策。整個系統在瀏覽器內運行，總資料量約5–10MB，不依賴任何3D引擎。

---

## 一、渲染架構

系統分為三個獨立渲染層，由下至上依序覆蓋。

### 背景層
- 30張 200×200 像素靜態圖片，代表不同視角、場景狀態或氣氛的擂台背景
- 遊戲啟動時全部預載入記憶體
- 根據遊戲事件（回合開始、擊倒、勝利）切換，不做連續動畫
- 壓縮後總量約 150–600KB

### 人物層
- 採用體素堆疊技術製造假3D效果
- 每個角色體積：40×30×20（寬×高×深）體素塊，約 24,000 個體素單位；雙人合計 48,000 個
- 渲染流程：斜角投影公式換算三維→二維座標 → 按Z軸深度排序 → 由遠到近繪製（近處自然遮擋遠處）
- 僅渲染表面體素，實際每幀繪製約 15,000–20,000 次
- 配合 0.3 秒刷新間隔，效能完全在瀏覽器可接受範圍內

### UI層
- 位於最頂層，包含血條、回合數、狀態提示
- 獨立於人物渲染之外，不受體素排序影響

### 全域刷新節奏
整個畫面每 **0.3 秒** 統一刷新一次，產生約 **3.3 FPS** 的刻意卡頓感，這正是像素格鬥遊戲的美學核心。

## 二、假3D人物系統

人物立體感來自兩個層面：**體素堆疊厚度** 與 **動作捕捉驅動的骨架變形**。

### 體素堆疊厚度

透過調整每個骨骼周圍的體素生成半徑（厚度參數）控制角色體型：

| 厚度值 | 效果 |
|--------|------|
| 1 | 純火柴人輪廓 |
| 4–5 | 標準體型拳手 |
| 6–8 | 壯漢體型 |

同一套骨架數據，只需改變厚度參數，即可生成不同體型的角色。

### 骨架數據（BVH格式）

- 採用業界標準 BVH 格式動作捕捉文件
- 每個 BVH 文件記錄約 25 個關節（頭、頸、肩、肘、腕、髖、膝、踝等）的每幀三維旋轉座標
- 系統讀取座標後，在相鄰關節之間插值，沿連線生成一串體素球，形成有體積的骨骼
- 所有骨骼組合即為完整人物體積

## 三、動作系統

### 動作分類

| 類別 | 動作清單 |
|------|----------|
| 拳法 | 直拳、勾拳、上勾拳、組合拳 |
| 腳法 | 前踢、側踢、旋踢、掃腿 |
| 防禦 | 格擋、閃躲、後退步 |
| 投技 | 抓頸、過肩摔、絆倒 |
| 倒地 | 前倒、後倒、膝倒 |
| 特殊 | 翻滾、跳躍攻擊、衝刺 |

### 動作切換邏輯
- 每個動作由若干幀組成，系統每 0.3 秒推進一幀，形成逐格動畫
- 動作切換時，在兩動作之間插值一幀，避免姿勢跳動
- 在 0.3 秒刷新頻率下，此過渡已足夠自然

## 四、套路神經網路

本質上是一個**動作狀態機加博弈樹**的結合，類似圍棋或五子棋的棋譜資料庫。

### 狀態節點結構

每個「狀態節點」代表一種當前情境，包含：
- 當前動作姿勢
- 雙方距離（貼身／近／中／遠／極遠，共五個區間）
- 自身重心方向
- 對手防禦狀態

每個節點連接多條「邊」，每條邊代表一個可能的下一步動作，附帶基礎權重和觸發條件。

### 套路資料庫規模
- 預計收入 200–300 條經典武打套路
- 每條套路記錄：動作序列、適用條件（起始姿勢、距離、對手狀態）、歷史成功率、對手反制動作
- 此結構讓系統不只知道「能做什麼」，還知道「做了之後對手可能怎麼應對」

---

## 五、動態權重系統

套路節點的邊權重不是固定數字，而是根據雙方屬性即時動態計算，由三個核心因素相乘得出最終權重。

### 力量差距
- 決定：破防能力、單次傷害倍率、擊退距離
- 力量佔優：勾拳、上勾拳等重拳路徑權重自動提高
- 力量較弱：系統自動引導走速攻和閃避路徑

### 敏捷速度
- 決定：連擊可用深度、閃避成功率
- 敏捷差距大：連擊第三動、第四動路徑自動開放
- 敏捷不足：這些邊被封鎖（代表速度跟不上，無法完成連擊）
- 速度差距達到閾值：快速角色獲得每回合額外一動

### 反應能力
- 決定：識破假動作能力、精準格擋時機窗口、反擊可用幀數
- 反應高：對手所有誘招和假動作路徑的權重大幅降低
- 反應低：誘招路徑的權重暴增，對手吃招機率極高

## 六、低幀率動態捕捉體素運鏡系統（Voxel Cine-System）

### 核心簡介

一套專為「假3D體素投影」架構開發的**非物理驅動演出引擎**。利用攝影機參數動畫（位置、角度、焦距）與渲染層級調度，在 3.3 FPS 的極低幀率下，實現媲美《一拳超人》、《鬼滅之刃》等高張力動漫的武打演出。

> **核心哲學：用極簡的幾何計算，換取極致的視覺衝擊。**

### 核心技術架構

**雙模投影轉換**
- **戰鬥態（斜角投影）：** 提供平穩、無死角的經典格鬥視角
- **演出態（透視投影）：** 模擬真實攝影機，產生「近大遠小」的壓迫感，用於必殺技特寫

**隱藏圖層調度（Cinematic Layer）**
- 必殺技觸發時，系統自動將人物體素數據抽離至獨立渲染層
- 進行高頻率的攝影機位移與視角旋轉，不干擾底層的戰鬥邏輯計算

**座標參數化腳本**
- 所有運鏡效果（震屏、子彈時間、衝擊幀）均編寫為輕量化參數物件
- 僅需改變攝影機與體素的座標偏移量，不需額外物理運算

### 四大運鏡特色

#### 1. 衝擊節奏補償（Impact Rhythm）
針對 3.3 FPS 的卡頓感進行優化。透過「全白衝擊幀（Impact Flash）」與「定格特寫」，讓玩家在每一幀的跳動中感受到紮實的受力反饋。

> **「不是因為卡頓而跳幀，而是為了打擊感而定格」**

#### 2. 體素動態形變（Voxel Mesh Deformation）
在命中瞬間直接修改受擊者的體素座標，依據攻擊力道產生「受擊塌陷」或「肉體扭曲」的視覺效果，還原動漫中拳拳到肉的拉扯感。

#### 3. 多重殘影追蹤（Ghost Trailing）
自動快照前 3–5 幀的體素座標，以遞減透明度進行重疊渲染。在高速移動或瞬殺動作時產生「老式街機」神韻的殘影，無需額外美術素材。

#### 4. 沉浸式攝影機綁定（Camera Parenting）
支持將攝影機直接綁定在「拳頭」或「飛劍」等動態關節上。角色出拳時，玩家視角隨拳頭同步衝擊，配合透視縮放，製造極強的視覺侵略性。

### 演出技法資料庫（部分收錄）

| 技法類別 | 代表風格 | 核心視覺特徵 | 套用情境 |
|----------|----------|--------------|----------|
| 壓制型 | 一拳超人流 | 定格 + 全白幀 + 體素噴發 | 終結技、重擊 |
| 速度型 | 浪客劍心流 | 運動模糊 + 座標瞬移 + 殘影 | 閃避反擊、瞬步 |
| 蓄力型 | 幽遊白書流 | 繞行攝影機 + 焦距拉伸 | 遠程法術、氣場爆發 |
| 戲劇型 | 鋼之鍊金術師流 | 畫面分割 + 仰角轉換 | 逆轉時刻、境界壓制 |

### 開發優勢
- **極低效能損耗：** 物理運算近乎為零，在各類瀏覽器環境皆能流暢運作
- **零素材成本：** 單純改變座標，不需額外的 2D 序列圖或 3D 模型
- **高度可擴展性：** 只要有 BVH 動捕數據與座標公式，即可無限擴展新的運鏡腳本

## 七、即時動態選單戰鬥系統（Instant-Tactics Combat System）

### 核心簡介

一套結合**格鬥遊戲的打擊感**與**策略遊戲的決策機制**的全新架構。取消複雜的搓招，轉而根據角色當前的物理狀態（位置、姿態、速度）即時生成「行動選單」。

### 核心操作邏輯：0.3 秒的策略窗口

系統每 0.3 秒根據當前情境刷新可執行選項，為玩家提供觀察與決策的空間。

- **基礎輸入：** 前後左右、拳、腳、必殺、跳躍、閃躲、跑步、防禦、蹲下
- **動態轉化：** 輸入組合或情境改變時，基礎輸入自動轉化為進階技能（例：跑步＋蹲下＝滑鏟）

### 情境感知狀態矩陣

系統透過三大維度即時定義角色狀態，作為選項生成依據：

| 維度 | 細項 |
|------|------|
| 空間狀態 | 與敵距離（遠/近/貼身）、相對方位（正面/背後/側面） |
| 物理動態 | 移動速度（靜止/跑/衝刺）、高度（滯空/站立/蹲下/倒地） |
| 技能熟練度 | 判定玩家是否已解鎖特定動作分支 |

### 動作失敗機制與教學引導

獨創「未學技能失敗」機制，將教學融入戰鬥：

- **功能運作：** 若玩家未學會「滑鏟」，在跑動中按下「蹲下」，人物將執行「摔倒」動作
- **資訊價值：** 失敗動作明確告知玩家「此處存在可發展的技能路徑」，引導解鎖如「鯉魚打滾」、「飛踢」等動作搭配

### 戰鬥演化：從基礎到套路

隨著戰鬥情境演變，選單內容產生質變：

1. **無人狀態（自由移動）：** 選項為普通位移與基礎動作
2. **交會狀態（戰鬥接觸）：**
   - **站立對峙：** 出現「拳、腳、防禦」等格鬥選項
   - **特殊姿態：** 如蹲下交會時，系統提供「昇龍拳（跳＋拳）」等進階選項
   - **貼身纏鬥：** 選項自動轉變為「鎖喉、過肩摔、八卦掌」

### 神經網路套路庫：三套路最低保證

神經網路必須在每 0.3 秒的交手瞬間，保證輸出至少 **3 個以上的有效應對套路**：

- **路數多樣性：** 如同五子棋棋譜預判，系統計算對手姿態與位置，提供「強攻、反制、迴避」三種不同傾向的選項
- **策略廣度：** 確保操作簡化不導致玩法單一，玩家必須在極短時間內判斷哪種路數能克制對手當前狀態

### 系統優勢

- **低門檻、高上限：** 操作不超過三個按鍵，對一般玩家極其友好；高端玩家可透過「位置誘引」與「姿態切換」控制神經網路生成的選項，具備深厚策略博弈空間
- **極致的可讀性：** 0.3 秒的節奏讓招式釋放變得清晰，玩家能明確感知「為何中招」或「如何反擊」

### 整合評估

這套系統透過「狀態驅動」取代了傳統的「肌肉記憶輸入」。它解決了格鬥遊戲對新手不友好的問題，並透過神經網路保證了每一場戰鬥的不可預測性與策略深度，是一款具備前沿性的 AI 驅動戰鬥框架。
